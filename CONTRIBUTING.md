# Contributing to Sparxstar Access Manager

Thank you for considering contributing to Sparxstar Access Manager!

## Development Setup

1. Clone the repository
2. Run `composer install` to install dependencies
3. Run `composer test` to ensure tests pass
4. Run `composer lint` to check code standards

## Code Standards

- Follow WordPress Coding Standards
- All PHP code must be compatible with PHP 7.4+
- Use meaningful variable and function names
- Document all classes and public methods
- Write tests for new functionality

## Pull Request Process

1. Ensure all tests pass: `composer test`
2. Ensure code follows standards: `composer lint`
3. Update documentation as needed
4. Update CHANGELOG.md with your changes
5. Submit pull request with clear description

## Testing

- Write unit tests for new functionality
- Ensure all existing tests pass
- Aim for high code coverage
- Test on both single-site and multi-site WordPress

## Reporting Issues

- Use GitHub Issues
- Provide clear description
- Include WordPress version
- Include PHP version
- Include steps to reproduce
